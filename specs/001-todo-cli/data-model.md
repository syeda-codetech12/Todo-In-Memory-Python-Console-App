# Data Model: Todo CLI Application

## Task Entity

**Name**: Task
**Description**: Represents a single todo item with id, title, description, and completion status

**Fields**:
- `id` (int): Unique incremental identifier for the task
- `title` (str): Required title of the task
- `description` (str, optional): Optional detailed description of the task
- `completed` (bool): Status indicator showing if the task is completed (default: False)

**Validation rules**:
- `id`: Must be a unique positive integer, auto-generated by the system
- `title`: Must be a non-empty string
- `description`: Optional, can be None or empty string
- `completed`: Must be a boolean value

**State transitions**:
- `pending` → `completed`: When task is marked as complete
- `completed` → `pending`: When task is marked as incomplete

## TaskList/TaskManager

**Name**: TaskManager
**Description**: In-memory collection of Task entities with operations to manage them

**Operations**:
- `add_task(title: str, description: str = None)`: Creates a new task with auto-generated ID
- `get_all_tasks()`: Returns all tasks in the collection
- `get_task_by_id(task_id: int)`: Returns a specific task by ID
- `update_task(task_id: int, title: str = None, description: str = None)`: Updates task details
- `delete_task(task_id: int)`: Removes a task from the collection
- `mark_task_complete(task_id: int)`: Sets task completion status to True
- `mark_task_incomplete(task_id: int)`: Sets task completion status to False

**Constraints**:
- Task IDs must be unique within the collection
- Task titles must not be empty when adding or updating
- Operations must fail gracefully if task ID doesn't exist
